# 기능 요청 자동 생성 AI — 3가지 수정 설계

- 작성일: 2026-06-05
- 대상 앱: **wedly-dev-request** (별도 저장소 `github.com/smlee-hash/wedly-dev-request`, main 브랜치)
- wedly-erp는 이 앱을 화면 안에 끼워 보여주기만 하므로 **건드리지 않음**

## 1. 배경 / 목적

기능 요청을 올릴 때, 자동 생성 AI가 (1) 별표(`**`) 기호를 본문에 남기고, (2) 핵심 외 군더더기를 붙이며, (3) 무조건 AI 구조화 단계를 강제해 사용자가 직접 쓴 글을 그대로 등록할 수 없다. 이 세 가지를 고친다.

## 2. 현재 동작과 문제 (증거)

| 문제 | 원인 (실제 코드) |
|---|---|
| ① 본문에 `**` `***` 노출 | AI 지시문이 일부러 `***제목***`, `**목적**` 마크다운 강조를 쓰도록 지시 — `app/api/dev-request/structure/route.ts:42-43`. 미리보기·편집칸·복사·노션에 그대로 노출 |
| ② 내용 과잉 | 지시문에 "간결" 항목은 있으나 약함(`route.ts:47`). 본문 첫 줄에 제목을 다시 적게 해 제목이 중복됨(`route.ts:42`) |
| ③ AI 없이 등록 불가 | 입력 화면 버튼이 `요청서 생성` 하나뿐 — `app/page.tsx:347`. 누르면 무조건 AI 구조화로 진입(`handleStructure`, `page.tsx:125`) |

참고: 등록 저장 화면(`page.tsx:364-438`)에는 **제목 입력칸이 아예 없다.** 제목은 AI가 정한 값이 화면에 안 보인 채 그대로 저장된다(`page.tsx:141, 177`).

## 3. 요구사항 (확정)

1. 본문에서 `**`/`***` 기호가 보이지 않게 한다.
2. 핵심만 간결하게 작성하게 한다.
3. 입력 화면에 버튼을 둘로 나눈다: `AI 구조화 후 등록`(기존) / `바로 등록`(신규, AI 생략).
   - `바로 등록`은 **확인 화면을 거치되 AI는 생략**한다(사용자 결정).
   - `바로 등록` 시 **제목은 입력 내용의 첫 줄에서 자동 제안**하고, 사용자가 수정 가능하게 한다(사용자 결정).

## 4. 설계

### ① 별표 제거 (지시문 + 안전장치 두 겹)

- **지시문 변경** (`route.ts` 프롬프트): 본문을 처음부터 별표 없는 깔끔한 평문으로 쓰게 한다.
  - "첫 줄에 `***제목***`" 규칙 삭제 → 제목은 별도 칸(`title`)에만 존재(중복 제거 = ② 효과).
  - "`**목적**:`" → "`목적:`" (별표 없는 평문 라벨).
  - 본문 형식: `목적: …` 1줄 + 번호 목록(`1.` `2.` …), 필요 시 하위 항목만 `- `. 표(테이블)·마크다운 강조 금지.
- **안전장치(서버)**: AI 응답을 돌려주기 직전, `title`과 `content`에서 별표 2개 이상 연속(`**`, `***`)을 제거한다. 정규식 `\*{2,}` 사용(단일 `*`는 보존해 "3*4" 같은 표현 손상 방지).
- **표시 화면**: 새 글은 평문이라 별표가 없음 → 미리보기·편집칸·복사·노션 모두 깨끗.
  - 등록 목록의 상세 펼침 표시기(`page.tsx:529-543`)는 **그대로 둔다**. 옛 글의 `**` 라벨을 굵게 변환하던 기존 로직은 옛 글 호환을 위해 유지(새 글은 평문이라 평문 분기로 자연 렌더).

### ② 내용 간결화 (지시문 강화)

- 지시문에 강한 문구 추가: "핵심 전달에 필요한 내용만. 부연 설명·수식어·미사여구 금지. 한 항목 한 줄."
- 제목 중복 제거(①에서 처리)로 군더더기 한 줄이 기본 사라진다.

### ③ '바로 등록' 버튼 + 제목칸 (AI 생략, 확인 화면 유지)

**입력 화면 버튼 (`page.tsx:346-351` 영역)**
- `AI 구조화 후 등록` — 기존 `handleStructure` 연결, 채워진(accent) 버튼.
- `바로 등록` — 신규 `handleDirectRegister` 연결, 외곽선(secondary) 버튼.
- `취소` — 기존 유지.
- 둘 다 입력 내용이 비면 비활성(기존 `disabled={!rawContent.trim()}` 동일 적용).

**신규 동작 `handleDirectRegister`**
- AI 호출 없음.
- `structuredContent = rawContent`(사용자 글 그대로).
- `structuredTitle = ` 입력의 **첫 비어있지 않은 줄**을 다듬어 사용(최대 60자로 자름). 자동 제안값(수정 가능).
- `category = "기타"`, `priority = "보통"`(기본값).
- `step = "review"`로 이동.

**확인 화면 변경 (`page.tsx:364-438`)**
- 맨 위에 **제목 입력칸 신규 추가**(항상 표시, 편집 가능, `structuredTitle`과 연결).
  - AI 경로: AI가 지은 제목이 채워진 채 표시 → 직접 수정 가능(덤 개선).
  - 바로 등록 경로: 첫 줄 자동 제안값이 채워짐.
- `노션에 등록` 버튼: 제목이 비어 있으면(공백 제외) 비활성.
- 카테고리 라벨의 "(AI 자동 분류)" 보조문구는 두 경로 공통이 되므로 **삭제**하고 라벨을 "카테고리"로 둔다.
- 등록 로직 `handleSubmit`은 기존 값(`structuredTitle/content/category/priority`)을 그대로 사용 → 추가 변경 최소.

## 5. 영향 파일

- `app/api/dev-request/structure/route.ts` — ①②(지시문 변경 + 별표 안전장치)
- `app/page.tsx` — ③(버튼 분리, `handleDirectRegister`, 제목 입력칸, 등록 버튼 잠금 조건)
- (확인) `wedly-erp/src/app/api/dev-request/structure/route.ts` — 끼워넣기 경로에서 미사용으로 추정. 사용처 확인 후 **죽은 코드면 범위 밖**(건드리지 않음).

## 6. 엣지 케이스 / 검증 포인트

- 입력이 빈 줄로 시작 → 첫 "비어있지 않은" 줄을 제목으로. 비어있지 않은 줄이 없으면 제목 빈 값 → 등록 버튼 잠금.
- 첫 줄이 매우 김 → 60자로 자른 제안.
- 사용자가 확인 화면에서 제목을 지움 → 등록 버튼 잠금.
- AI가 지시 무시하고 별표 출력 → 서버 안전장치가 제거.
- 단일 `*`(예: "3 * 4")는 보존(안전장치가 2개 이상만 제거).
- 바로 등록은 네트워크/AI 호출이 없으므로 AI 지연·오류 영향 없음.
- 첨부 파일 흐름은 두 경로 공통(`handleSubmit`에서 업로드) — 바로 등록도 첨부 유지.

## 7. 배포

- 빌드 확인: `npm run build` (= `prisma generate && next build`) 성공.
- `git commit` + `git push origin main` → 자동 배포.
- 업데이트 알림: 이 앱 자체 시스템(`/api/app-updates`)으로 등록(카테고리 `improve`, 해당 페이지 경로). wedly-erp의 deploy-check.sh는 사용하지 않음.

## 8. 범위 밖 (이번에 하지 않음)

- 이미 노션에 등록된 옛 글의 별표 일괄 정리(데이터 변경 위험 → 새 글부터 적용).
- 기존 파일의 다른 색상/디자인 토큰 정리(이번 요청과 무관).
- wedly-erp 쪽 변경.
